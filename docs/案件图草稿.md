# Case Graph Draft Outcome

Updated: 2026-04-25

This draft has been resolved into the implemented canonical case graph.

Current node kinds:

- problem
- hypothesis
- blocker
- repair_attempt
- evidence_ref
- reusable evidence in the evidence pool

Current structural rules:

- problem is the only root.
- hypotheses explain or refine the problem.
- blockers represent missing access, missing evidence, or unresolved conflicts.
- repair attempts require a confirmed hypothesis.
- evidence references attach reusable evidence to a hypothesis or repair attempt.

Use `docs/plans/2026-04-18-canonical-case-graph-model.md` for the maintained
model and `packages/domain` for executable rules.
